# Code of Conduct

UIverse is a beginner-friendly project. We want every contributor — first
PR or hundredth — to feel welcome.

## Our standards

- **Be respectful and welcoming** to all contributors, regardless of
  experience level.
- **Give constructive feedback** on PRs. Explain _why_, not just _what_.
- **Help beginners.** Everyone starts somewhere — if you see someone stuck,
  offer a pointer rather than just a downvote or rejection.
- **No spam PRs.** Whitespace-only changes, typo-fix-only PRs with no
  substance, or PRs opened purely to farm contribution counts will be
  closed without merge.
- **Stay on topic.** Keep issue and PR discussions focused on the technical
  matter at hand.
- **No harassment, discrimination, or personal attacks** of any kind, in
  any project space (issues, PRs, discussions, comments).

## Unacceptable behavior

- Insulting, derogatory, or dismissive comments about a contributor's
  skill level or code
- Harassment, public or private
- Publishing others' private information without consent
- Sustained disruption of discussions or review threads

## Enforcement

Project maintainers are responsible for clarifying and enforcing these
standards. Maintainers may remove comments, close PRs/issues, or
temporarily/permanently ban contributors who violate this code of conduct.

If you experience or witness unacceptable behavior, please open a private
report by contacting a maintainer directly, or via GitHub's report feature
on the comment/PR in question.

## Scope

This Code of Conduct applies within all project spaces — issues, pull
requests, discussions, and any other UIverse community channels — and also
applies when an individual is representing the project in public spaces.
